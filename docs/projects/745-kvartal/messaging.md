# Messaging: 745-kvartal

Этот документ фиксирует текущую смысловую рамку лендинга `745-kvartal`, чтобы оффер, JTBD-логика, CTA и правила вординга были управляемым артефактом в репозитории, а не оставались только в коде страницы или в истории обсуждений.

## Messaging Role

Документ нужен для того, чтобы команда могла быстро ответить на три вопроса: кому адресован лендинг, какую работу для аудитории выполняет продукт и какими формулировками это допустимо выражать на странице.

## Primary Audience

Текущий основной адресат hero и большей части страницы — потенциальный покупатель квартиры в ЖК `745-kvartal`.

| Поле | Зафиксированное решение |
|---|---|
| Primary audience | Потенциальный покупатель ЖК |
| Secondary audience | Топ-менеджер девелопера, который видит лендинг как пример переупаковки |
| Conversion focus | Для покупателя — интерес к продукту и погружение в смысл проекта; для девелопера — контакт через финальную B2B-форму |

## JTBD Frame

JTBD-логика для страницы должна описывать не абстрактную покупку недвижимости, а переход человека в более соразмерный ему жизненный сценарий.

> Покупатель не просто выбирает квартиру. Он выбирает среду, в которой можно жить в своём масштабе, без лишней демонстрации, шума и навязанных сценариев.

| Компонент | Формулировка |
|---|---|
| Functional job | Найти квартиру бизнес-класса в тихом центре с понятной приватностью, качеством и ограниченным предложением |
| Emotional job | Перестать жить в компромиссе между статусом, удобством и внутренним ощущением своего места |
| Social job | Выбрать дом, который считывается как зрелый выбор, а не как показное потребление |

## Current Offer Direction

Hero-оффер должен быть направлен на клиента ЖК, а не на девелопера. Девелоперский B2B-сценарий разрешён только в финальном блоке формы.

| Элемент | Правило |
|---|---|
| Hero offer | Только для потенциального покупателя ЖК |
| Hero CTA | Мягкий CTA уровня ознакомления, а не жёсткого давления |
| Final form | B2B-сценарий для топ-менеджера застройщика |

## Current CTA Rules

| Зона страницы | Текущее правило |
|---|---|
| Hero CTA | Использовать формулировку **«Ознакомиться с предложением»** или близкую по смыслу мягкую навигационную кнопку |
| Mid-page CTA | Поддерживать переход к деталям, а не ломать настроение страницы агрессивной продажей |
| Final CTA | Выводить девелопера на контакт по теме переупаковки позиционирования |

## Wording Principles

Текущий лендинг должен говорить в спокойной, собранной и статусной манере. Вординг не должен звучать как массовая девелоперская реклама.

| Принцип | Что это значит на практике |
|---|---|
| Спокойное достоинство | меньше крика, больше уверенности и внутренней собранности |
| Без показной роскоши | не использовать штампы про премиальность, исключительность и успех ради демонстрации |
| Смысл прежде лозунга | каждый сильный тезис должен быть связан с реальным качеством продукта |
| Рациональная опора | эмоциональные формулировки должны поддерживаться конкретикой |
| Ограничение B2B-слоя | девелоперский тон не должен протекать в hero и основной пользовательский сценарий |

## Anti-Patterns

| Нежелательная формулировка | Почему её нужно избегать |
|---|---|
| Слишком общий лайфстайл-лозунг без связи с продуктом | теряется доверие и JTBD-опора |
| Агрессивный продающий CTA в hero | разрушает тональность и снижает ощущение класса |
| Смешение покупателя и девелопера в одном оффере | ломает фокус страницы |
| Перегрузка абстракциями без конкретики | страница начинает звучать как бренд-манифест без продукта |

## Maintenance Rule

Если меняется hero-оффер, JTBD-логика, основная CTA-стратегия, тональность страницы или граница между B2C- и B2B-слоем, это изменение должно обновляться не только в коде, но и в этом документе.
