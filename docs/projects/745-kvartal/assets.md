# Assets: 745-kvartal

Этот документ фиксирует, какие визуальные ассеты относятся к лендингу `745-kvartal`, где они находятся в репозитории и как ими нужно управлять, чтобы проект не превращался в неструктурированное хранилище изображений.

## Asset Role

Реестр ассетов нужен для трёх задач: быстро находить используемые изображения, отличать продакшн-ассеты от референсов и понимать, какие файлы действительно являются частью лендинга, а какие были лишь временными рабочими материалами.

## Storage Principle

Все ассеты лендинга должны храниться по `slug` проекта и быть отделены от временных файлов.

| Тип | Правило хранения |
|---|---|
| Production assets | Хранятся в `public/assets/745-kvartal/` |
| Original references | Хранятся во вложенной структуре, например `public/assets/745-kvartal/original/` |
| Temporary working files | Не должны оставаться в репозитории после завершения работы |
| External references | Должны быть зафиксированы в этом документе ссылкой или текстовым описанием источника |

## Known Assets

На текущий момент для проекта зафиксирован как минимум один важный визуальный артефакт.

| Asset | Path | Role | Status |
|---|---|---|---|
| Current positioning screenshot | `public/assets/745-kvartal/original/current_positioning.webp` | Скриншот текущего позиционирования девелопера, используемый в финальном B2B-блоке как контрастный материал | Active |

## Classification Rules

| Категория | Как трактовать |
|---|---|
| Active | Ассет реально используется на странице или является обязательной частью её сценария |
| Reference | Ассет нужен как референс для переупаковки, но не обязательно рендерится на странице |
| Archived | Ассет больше не используется, но сохранён как часть проектной истории |
| Temporary | Временный файл для работы; не должен попадать в постоянную структуру репозитория |

## Naming Rules

Названия файлов должны быть семантическими и читаемыми. Имя должно позволять понять роль ассета без открытия самого файла.

| Хорошая практика | Пример |
|---|---|
| Указывать контекст проекта | `745-kvartal/...` |
| Отражать назначение файла | `current_positioning.webp` |
| Избегать случайных названий и upload-артефактов | не использовать `image-final-2-new.webp`, `pasted_file_123.webp` |

## Maintenance Rule

Если в лендинг добавляется новый production-ассет, референсное изображение, исходник для контрастного сравнения или архивируемый визуальный материал, запись о нём должна появиться в этом документе. Если файл был временным и больше не нужен, он должен быть удалён, а не оставлен в репозитории без статуса.
