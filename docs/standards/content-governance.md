# Content Governance Standard

Этот документ определяет, какие текстовые артефакты должны попадать в репозиторий, а какие считаются временными рабочими материалами.

## Главный принцип

В GitHub хранится только то, что обладает долгосрочной проектной ценностью: объясняет решение, фиксирует процесс, поддерживает повторяемость или влияет на эксплуатацию лендинга.

## Что обязательно хранить

| Категория | Где хранить |
|---|---|
| Брифы и постановки | `docs/projects/<slug>/brief.md` |
| JTBD, сегменты, логика аудитории | `docs/projects/<slug>/audience-and-jtbd.md` или `messaging.md` |
| Согласованный вординг | `docs/projects/<slug>/messaging.md` |
| Аудиты и выводы | `docs/projects/<slug>/audit.md` |
| История смысловых изменений | `docs/projects/<slug>/change-log.md` |
| Важные локальные решения | `docs/projects/<slug>/decisions.md` |

## Что не хранить

В репозиторий не должны попадать сырые заметки LLM, черновые выжимки чатов, промежуточные scratch-файлы, одноразовые размышления и временные comparison notes, если они не были консолидированы в устойчивый документ.

## Правило консолидации

Если временный материал содержит ценное решение, это решение должно быть перенесено в один из системных документов. Сам временный файл после этого не является обязательной частью репозитория.
