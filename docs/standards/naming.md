# Naming Standard

Этот стандарт задаёт правила именования файлов, директорий и идентификаторов в проекте **landing-factory**, чтобы при росте числа лендингов навигация оставалась предсказуемой.

## Базовый принцип

Основной идентификатор проекта — **slug**. Один лендинг должен иметь один и тот же slug во всех слоях системы.

| Сущность | Формат | Пример |
|---|---|---|
| Лендинг | kebab-case slug | `745-kvartal` |
| Страница | `src/pages/<slug>.astro` | `src/pages/745-kvartal.astro` |
| Папка документации | `docs/projects/<slug>/` | `docs/projects/745-kvartal/` |
| Папка ассетов | `public/assets/<slug>/` | `public/assets/745-kvartal/` |

## Правила именования

Имена должны быть короткими, семантическими и стабильными. Не допускаются пробелы, кириллица в путях, версии вида `final-final`, а также имена, завязанные на временный контекст сессии.

| Допустимо | Недопустимо |
|---|---|
| `change-log.md` | `new version 2.md` |
| `audience-and-jtbd.md` | `целевая аудитория.md` |
| `current-positioning.webp` | `скриншот 1.webp` |

## Именование документов

Внутри `docs/projects/<slug>/` следует использовать фиксированный словарь ролей документов. Если возникает новый тип файла, он должен быть либо добавлен в стандарт, либо оформлен как локальное исключение в `decisions.md`.
