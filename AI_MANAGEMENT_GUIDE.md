# Руководство по управлению сайтом через ИИ-ассистента (Manus / Claude)

Это руководство предназначено для работы с проектом **Landing Factory** (DevPartners) через ИИ-ассистентов. Проект спроектирован так, чтобы ассистент мог выполнять 100% задач по управлению сайтом: от правки опечаток до массового создания лендингов и настройки инфраструктуры.

Главное правило при работе с ИИ: **всегда давайте ему точный контекст**. Ассистент должен понимать, где лежат данные, как устроена архитектура и что именно нужно сделать.

---

## 1. Архитектура проекта (Контекст для ИИ)

Когда вы начинаете новую сессию с ассистентом, полезно дать ему вводную команду, чтобы он сразу понял структуру проекта.

**Скопируйте и отправьте этот промпт в начале работы:**
> «Мы работаем с проектом Landing Factory (Astro + Netlify). 
> Архитектура:
> 1. Данные лендингов лежат в `src/data/landings/*.json`.
> 2. Страницы генерируются в `src/pages/*.astro` (каждая страница импортирует свой JSON и нужный шаблон).
> 3. Шаблоны лежат в `src/templates/` (offer801, aesthetes, stay-but-grow, parental-capital).
> 4. Картинки лежат в `public/assets/`. Дефолтные картинки шаблонов лежат в `public/assets/{имя_шаблона}/original/`.
> 5. Редиректы настроены в `public/_redirects`.
> 6. Реестр всех лендингов ведётся в `REGISTRY.md`.
> Пожалуйста, изучи структуру папок перед выполнением задач.»

---

## 2. Создание нового лендинга

Создание лендинга состоит из трёх шагов: создание JSON-данных, создание Astro-страницы и генерация скриншота позиционирования.

**Промпт для создания лендинга:**
> «Создай новый лендинг для ЖК [Название ЖК] от застройщика [Название застройщика] в городе [Город].
> 1. Используй шаблон `[offer801 / aesthetes / stay-but-grow / parental-capital]`.
> 2. Создай файл `src/data/landings/[slug].json`, взяв за основу структуру существующего лендинга на этом шаблоне. Адаптируй все тексты (заголовки, карточки, триггеры) под этот ЖК.
> 3. В JSON-файле пути к картинкам (`images.hero`, `images.aha1` и т.д.) должны вести на дефолтные картинки шаблона: `/assets/[имя_шаблона]/original/...`.
> 4. Путь к скриншоту позиционирования укажи как `/assets/[slug]/current_positioning.png`.
> 5. Создай файл `src/pages/[slug].astro`, который импортирует этот JSON и нужный шаблон.
> 6. Добавь запись о новом лендинге в `REGISTRY.md`.
> 7. Сделай коммит и пуш.»

**Промпт для добавления скриншота:**
> «Я загрузил скриншот сайта застройщика [имя файла]. Помести его в папку `public/assets/[slug]/` под именем `current_positioning.png` (или .webp). Убедись, что путь в JSON-файле совпадает. Сделай коммит и пуш.»

## 3. Массовое создание лендингов (Батчинг)

Ассистент отлично справляется с созданием десятков лендингов за раз, если дать ему таблицу с данными.

**Промпт для массового создания:**
> «Во вложении таблица с новыми застройщиками и ЖК. Нам нужно создать лендинги для всех строк.
> 1. Проанализируй таблицу. Для каждого ЖК определи подходящий шаблон (offer801 для стандарта, aesthetes для премиума, stay-but-grow для семейных, parental-capital если есть акцент на маткапитал).
> 2. Напиши скрипт на Python, который сгенерирует JSON-файлы для всех ЖК, адаптируя тексты под каждый проект.
> 3. Скрипт также должен сгенерировать `.astro` файлы в `src/pages/`.
> 4. Обнови `REGISTRY.md` списком всех новых лендингов.
> 5. Сделай билд (`npm run build`), чтобы проверить отсутствие ошибок, затем коммит и пуш.»

---

## 4. Редактирование текстов на существующих лендингах

Если нужно поправить опечатку, изменить оффер или переписать блок на конкретном лендинге.

**Промпт для точечной правки:**
> «В лендинге `[slug]` нужно изменить текст.
> Открой его JSON-файл в `src/data/landings/`.
> Найди блок `[название блока, например: hero.h1 или responseCards]`.
> Замени текущий текст на: "[Новый текст]".
> Проверь, что JSON остался валидным, сделай коммит и пуш.»

**Промпт для массовой правки:**
> «Нам нужно изменить текст на кнопке (ctaText) во всех лендингах, которые используют шаблон `offer801`.
> Напиши скрипт, который пройдёт по всем JSON-файлам, проверит поле `template`, и если это `offer801`, заменит `hero.ctaText` на "Получить разбор проекта".
> Сделай коммит и пуш.»

---

## 5. Управление редиректами (UTM-метки и короткие ссылки)

Все редиректы хранятся в файле `public/_redirects`. Netlify поддерживает редиректы по query-параметрам (UTM-меткам).

**Промпт для добавления короткой ссылки:**
> «Добавь новый алиас в `public/_redirects`. Ссылка `/offer50` должна вести на лендинг `/[slug]/`. Код ответа 200. Сделай коммит и пуш.»

**Промпт для добавления UTM-редиректов из таблицы:**
> «Во вложении таблица рассылки. В ней есть названия компаний и ссылки с UTM-метками (параметр `utm_content`).
> 1. Сопоставь компании из таблицы с существующими лендингами в проекте.
> 2. Добавь правила в `public/_redirects` в формате: `/offer utm_content=[код] /[slug]/ 302`.
> 3. Убедись, что старые правила не удалены. Сделай коммит и пуш.»

---

## 6. Изменение дизайна (Правка шаблонов)

Если нужно изменить вёрстку, добавить новый блок или поменять стили — это делается в файлах `.astro` и `.css`.

**Промпт для правки шаблона:**
> «Мне нужно изменить дизайн карточек в шаблоне `stay-but-grow`.
> 1. Изучи файл `src/templates/StayButGrowTemplate.astro` и связанный с ним CSS (обычно импортируется в блоке `<style>`).
> 2. [Опишите, что нужно изменить, например: "Сделай углы карточек более скругленными (border-radius: 24px) и увеличь отступ внутри"].
> 3. Внеси изменения, сделай `npm run build` для проверки, затем коммит и пуш.»

---

## 7. Решение проблем и дебаггинг

Если сайт не собирается или выглядит неправильно, ассистент может сам найти и исправить ошибку.

**Промпт для починки сборки:**
> «При деплое на Netlify (или при `npm run build`) возникает ошибка.
> Запусти билд локально, проанализируй вывод ошибки. Скорее всего проблема в невалидном JSON-файле (пропущена запятая или кавычка) или в отсутствии нужного поля, которое ожидает шаблон.
> Найди сломанный файл, исправь его, проверь билд снова и запушь.»

**Промпт для починки "пропавших" картинок:**
> «На лендинге `[slug]` не отображаются картинки в блоке [название блока].
> Проверь JSON-файл этого лендинга: какие пути указаны в секции `images`?
> Проверь через `ls`, существуют ли реально эти файлы в папке `public/assets/`.
> Если файлов нет, исправь пути в JSON так, чтобы они вели на дефолтные картинки соответствующего шаблона. Запушь исправления.»
